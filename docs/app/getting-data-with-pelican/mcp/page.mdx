# Using Pelican with AI Assistants via MCP

The Pelican Model Context Protocol (MCP) server enables AI assistants like Claude Code (CLI tool), VS Code Copilot, and other MCP-compatible tools to download files, inspect metadata, and list directories from Pelican federations. This allows you to use natural language to interact with Pelican data directly from your terminal or AI-powered development environment.

## What is MCP?

The [Model Context Protocol](https://modelcontextprotocol.io/) is an open standard that allows AI assistants to connect to external tools and data sources. By running a Pelican MCP server, your AI assistant can:

- Download objects from Pelican federations using natural language requests
- Get metadata about objects (size, modification time, checksums)
- List directory contents
- Work with **public namespaces** and **protected namespaces** (with limitations - see below)

## Important Limitations

The Pelican MCP server has important authentication limitations you should be aware of:

### ✅ What Works
- **Public namespaces**: Full support for downloading, listing, and inspecting public data
- **Protected namespaces** (after CLI authentication): May work if you authenticate via the Pelican CLI first
- **Explicit token files**: Can provide token files in your prompts

### ❌ What Doesn't Work
- **First-time authentication to protected namespaces**: The MCP server cannot perform interactive OAuth/OIDC flows
- **Browser-based login**: Cannot open browsers or redirect to SSO providers (CILogon, university logins, etc.)
- **Interactive token acquisition**: Cannot prompt for passwords or handle multi-step authentication

**Bottom line**: For protected namespaces, you must authenticate using the Pelican CLI first, then the MCP server may be able to reuse cached tokens. See [Working with Protected Namespaces](#working-with-protected-namespaces) for details.

## Before Starting

### Assumptions

This guide assumes:

- You have Pelican installed on your system with the `pelican` command accessible in your PATH
- You're familiar with [Pelican's command line client](../client/page.mdx) and basic concepts
- You have an MCP-compatible AI assistant installed (Claude Code or VS Code with GitHub Copilot)
- You're comfortable editing JSON configuration files

### Verify Pelican Installation

Ensure the Pelican MCP server is available:

```bash copy
pelican mcp --help
```

You should see output describing the `pelican mcp serve` command. If not, refer to the [Pelican installation docs](../../install.mdx) to install or update Pelican.

## Setting Up with Claude Code

Claude Code is Anthropic's official AI-powered CLI tool that natively supports MCP servers, making it easy to integrate Pelican into your terminal workflow.

### 1. Locate Your Configuration File

The Claude Code MCP configuration file location depends on your operating system:

**macOS**:
```bash
~/Library/Application Support/Claude/claude_mcp_config.json
```

**Windows**:
```bash
%APPDATA%\Claude\claude_mcp_config.json
```

**Linux**:
```bash
~/.config/claude/claude_mcp_config.json
```

### 2. Add Pelican MCP Server

Edit the configuration file and add the Pelican MCP server to the `mcpServers` section:

```json copy
{
  "mcpServers": {
    "pelican": {
      "command": "pelican",
      "args": ["mcp", "serve"]
    }
  }
}
```

If you already have other MCP servers configured, simply add the `"pelican"` entry alongside them:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/username/Desktop"]
    },
    "pelican": {
      "command": "pelican",
      "args": ["mcp", "serve"]
    }
  }
}
```

### 3. Configuration is Automatic

After saving the configuration file, Claude Code will automatically load the Pelican MCP server when you start a new session. No restart is required - the MCP server is loaded on-demand.

### 4. Verify Connection

Start a Claude Code session in your terminal. When you mention Pelican operations, Claude Code will automatically connect to the MCP server. You can verify the connection by asking Claude to list available tools - "pelican" should appear in the connected MCP servers.

### 5. Use Natural Language to Download Data

You can now ask Claude to download Pelican objects using natural language:

```
Download the file pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/test.txt to ~/Downloads/test.txt
```

Claude will use the `pelican_download` tool to fetch the file and confirm when it's complete.

## Setting Up with VS Code

VS Code with GitHub Copilot can also use MCP servers to enhance its capabilities.

### 1. Install Required Extensions

Ensure you have:
- [GitHub Copilot extension](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot)
- MCP support (available in recent Copilot versions)

### 2. Configure MCP Settings

Open VS Code settings (`Cmd/Ctrl + ,`) and search for "MCP". Add the Pelican server to your MCP configuration:

**Option A: Via Settings UI**

1. Search for "github.copilot.mcp.servers"
2. Click "Edit in settings.json"
3. Add the Pelican configuration

**Option B: Direct settings.json Edit**

Open your VS Code settings.json (`Cmd/Ctrl + Shift + P` → "Preferences: Open User Settings (JSON)") and add:

```json copy
{
  "github.copilot.mcp.servers": {
    "pelican": {
      "command": "pelican",
      "args": ["mcp", "serve"]
    }
  }
}
```

### 3. Reload VS Code

After saving the configuration, reload VS Code:
- Press `Cmd/Ctrl + Shift + P`
- Type "Developer: Reload Window"
- Press Enter

### 4. Use Copilot Chat with Pelican

In the Copilot Chat panel, you can now request Pelican operations:

```
@workspace Download pelican://osg-htc.org/pelicanplatform/test/hello-world.txt to the current workspace
```

Copilot will use the Pelican MCP server to download the file.

## Available Tools

Once configured, your AI assistant has access to three Pelican tools:

### pelican_download

Downloads objects from Pelican URLs to your local filesystem.

**Example prompts**:
- "Download pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/test.txt to /tmp/test.txt"
- "Get the hello-world.txt file from pelican://osg-htc.org/pelicanplatform/test/ and save it locally"
- "Download the entire validation directory recursively from pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/"

**Parameters**:
- `source`: The Pelican URL to download from
- `destination`: Local path where the object should be saved
- `recursive`: (optional) Set to true to download directories recursively
- `token`: (optional) Authentication token for protected namespaces

### pelican_stat

Gets metadata about a Pelican object including size, modification time, and checksums.

**Example prompts**:
- "What's the size of pelican://osg-htc.org/pelicanplatform/test/hello-world.txt?"
- "Get information about the validation test file"
- "Show me the metadata for pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/test.txt"

**Parameters**:
- `url`: The Pelican URL to inspect
- `token`: (optional) Authentication token for protected namespaces

### pelican_list

Lists the contents of a Pelican directory.

**Example prompts**:
- "List the contents of pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/"
- "What files are in the pelicanplatform test directory?"
- "Show me all objects under pelican://osg-htc.org/pelicanplatform/test/"

**Parameters**:
- `url`: The Pelican URL of the directory to list
- `token`: (optional) Authentication token for protected namespaces

## Working with Protected Namespaces

> **⚠️ Important Authentication Limitations**
>
> The Pelican MCP server runs as a non-interactive background process and **cannot**:
> - Open a browser for OAuth/OIDC authentication flows
> - Prompt for user input or passwords
> - Complete interactive CILogon or university SSO login flows
>
> This means you **cannot authenticate to a protected namespace for the first time** via MCP.
>
> **For protected namespaces, you must authenticate using the Pelican CLI first**, which will cache tokens that the MCP server may be able to reuse. Alternatively, you can provide token files explicitly.

### Authentication Workflow for Protected Namespaces

#### Step 1: Authenticate via Pelican CLI

Before using the MCP server with protected namespaces, authenticate using the command line:

```bash copy
pelican object get pelican://<federation-url>/<protected-namespace>/<path> <destination>
```

This will:
1. Guide you through the OAuth/OIDC login flow in your browser
2. Store the authentication token in your Pelican token wallet (`~/.config/pelican/credentials/`)

See [Getting Protected Objects](../client/page.mdx#get-a-protected-object-from-your-federation) for detailed CLI authentication instructions.

#### Step 2: Use MCP Server (Token Reuse)

After CLI authentication, the MCP server **may** be able to reuse cached tokens automatically:

```
Download pelican://my-federation.org/protected/data.txt to ~/Downloads/
```

> **Note**: Automatic token reuse from the wallet is not yet fully verified. If this doesn't work, use the explicit token method below.

#### Alternative: Provide Tokens Explicitly

If automatic token reuse doesn't work, or if you have a standalone token file, provide it explicitly:

```
Download pelican://my-federation.org/protected/data.txt using the token from ~/my-token.jwt
```

The AI assistant will pass this to the `pelican_download` tool's `token` parameter.

### Obtaining Standalone Token Files

To get a standalone token file for use with MCP:

```bash copy
# Authenticate and export token
pelican object get pelican://<federation-url>/<protected-namespace>/<path> /tmp/test \
  --token-location ~/my-pelican-token.jwt
```

Then use this token file with your AI assistant as shown above.

## Example Workflows

### Downloading from Public Namespaces

```
User: Download pelican://osg-htc.org/pelicanplatform/test/hello-world.txt to ~/Downloads/

AI: I'll download that file for you. [Uses pelican_download tool]
Successfully downloaded to ~/Downloads/hello-world.txt (27 bytes)
```

### Exploring Available Data

```
User: What data is available in the pelican://osg-htc.org/pelicanplatform/test/ directory?

AI: Let me check that for you. [Uses pelican_list tool]

The directory contains:
- hello-world.txt (27 bytes)
- sample-data.csv (1.2 MB)
- README.md (543 bytes)
```

### Checking File Metadata Before Download

```
User: Before I download pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/test.txt,
can you tell me how large it is?

AI: [Uses pelican_stat tool]

The file is 27 bytes, last modified on 2024-03-15. It has MD5 checksum: a1b2c3d4...
Since it's very small, would you like me to download it?

User: Yes please, to /tmp/

AI: [Uses pelican_download tool]
Downloaded to /tmp/test.txt
```

## Troubleshooting

### MCP Server Not Connecting

If your AI assistant shows the Pelican MCP server as disconnected:

1. **Verify Pelican is in PATH**:
   ```bash
   which pelican
   ```
   If this returns nothing, add Pelican to your PATH or use the full path in your MCP configuration.

2. **Test MCP server manually**:
   ```bash
   pelican mcp serve
   ```
   You should see "Pelican MCP server started" in the output. Press Ctrl+C to exit.

3. **Check configuration syntax**:
   Ensure your JSON configuration is valid. Use a JSON validator if needed.

4. **Restart your AI assistant completely**:
   Make sure you fully quit (not just close) and relaunch.

### Permission Errors

If you get permission errors:

```bash
chmod +x $(which pelican)
```

### Downloads Fail

If downloads fail with authentication errors:

1. Ensure you've authenticated for protected namespaces using the CLI first
2. Check that tokens are stored in your Pelican token wallet
3. For public namespaces, verify the URL is correct using the CLI:
   ```bash
   pelican object get pelican://... /tmp/test
   ```

### AI Assistant Doesn't Use Pelican Tools

If your AI assistant isn't using the Pelican tools:

1. **Be explicit**: Say "Use the pelican_download tool to..."
2. **Check MCP connection**: Verify the server shows as connected
3. **Provide complete Pelican URLs**: Include the full `pelican://` URL

## Advanced Configuration

### Custom Pelican Installation Path

If Pelican is not in your system PATH, specify the full path:

```json
{
  "mcpServers": {
    "pelican": {
      "command": "/opt/pelican/bin/pelican",
      "args": ["mcp", "serve"]
    }
  }
}
```

### Debug Mode

To enable debug logging for troubleshooting:

```json
{
  "mcpServers": {
    "pelican": {
      "command": "pelican",
      "args": ["mcp", "serve", "--debug"]
    }
  }
}
```

Logs will appear in your AI assistant's MCP server logs.

### Environment Variables

You can pass environment variables to configure the MCP server:

```json
{
  "mcpServers": {
    "pelican": {
      "command": "pelican",
      "args": ["mcp", "serve"],
      "env": {
        "PELICAN_FEDERATION_URL": "https://my-federation.org"
      }
    }
  }
}
```

## Best Practices

### Use Descriptive Prompts

Instead of:
```
Download the file
```

Use:
```
Download pelican://osg-htc.org/pelicanplatform/test/hello-world.txt to ~/Downloads/hello.txt
```

### Verify Before Large Downloads

For large files or directories, ask for metadata first:

```
How large is pelican://osg-htc.org/ospool/data/large-dataset/ recursively?
```

Then download if appropriate.

### Organize Downloaded Files

Specify organized destination paths:

```
Download pelican://osg-htc.org/ospool/uc-shared/public/OSG-Staff/validation/test.txt
to ~/pelican-data/validation/test.txt
```

The MCP server will create directories as needed.
